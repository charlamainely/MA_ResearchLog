Literature

The Reflective Practitioner

Donald A. Schön Key concepts: reflection-in-action, reframing, repertoire, sketching, affirmation logic

Work

Schön, Donald A. The Reflective Practitioner: How Professionals Think in Action. (Cite your edition: publisher, year).

Key Takeaways

  • What counts as a practice (p.60): practices repeat situation-types; risk of becoming “over-learned” → reflection counters routine blindness.
  • Reflection-in-action loop (pp.129–133): in uncertain/unique cases, avoid default “standard techniques”; instead reframe → experiment → re-appreciate as unintended outcomes appear (“appreciation → action → re-appreciation”).
  • Reframing prompts (p.133): solvability, desirability (“Do I like what I get?”), coherence, values congruence, and “Have I kept inquiry moving?”.
  • Logic of affirmation (p.155): practice privileges change; failed moves trigger critique of the implicit theory → restructure → test a new move.
  • Sketchpad as “virtual world” (pp.158–159): sketching enables rapid, low-constraint trials and noticing unintended results; insights must transfer to the built world.
  • Practice as research (p.308): practitioners can be reflective researchers amid uncertainty and conflict.
  • Frames analysis steer action (p.309): frames set strategy and encode values.
  • Repertoire building research (p.315): accumulated examples strengthen in-action judgment and reframing.

Reflections / Critical Notes

In this MA, my practice is game design + prototyping + user testing toward preserving ICH. Schön’s loop maps cleanly onto my process: start with a problem framing, test a specific mechanic observe consequences, and look out for unintended outcomes to refocus the next iteration. To avoid “over-learning,” I’ll log new insights from each playtest and use them as evidence against my current design theory.

My position as a game designer drawn to oral traditions of Vietnam shapes how I read Schön: I naturally prioritize games mechanics and entertainment values, which could undermine cultural nuances if not careful. I’m also early-career; novelty to the practice makes reframing feel “easier,” but limited repertoire can slow sense-making. I will therefore (a) state my values frame (community engagement, storytelling practices, accessibility) in each prototype note, and (b) expand a repertoire of interactive design/mechanics to strengthen my in-action choices.

Integrating Reflection-in-Action in my workflow

  • Frame statement per sprint (1–2 lines: aim + values + audience).
  • Affirmation logic log: when a move fails, write the implicit theory → revise → specify the next move to test.
  • RiA questions (p.133) 'Can I solve the problem I have set?', 'Do I like what I get when I solve this problem?', 'Have I made the situation coherent?', 'Have I made it congruent with my fundamental values and theories?', 'Have I kept inquiry moving?'
  • Transfer check for sketches: pair each “virtual” sketch with a micro-prototype and 2–3 quick user checks.
