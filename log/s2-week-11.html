Reflection

After the first round of user test, the anaglyph puzzle is disappointing me quite a lot as it did not generate the amount of excitement as I expected. The main complaint has been from the lens not working well and the word search being too challenging without clues. So these are some notes I want to address in the next iteration of this prototype.

Another development from this week is the printer installation, since I have managed to make the printer print by manually sending commands from my laptop. The next stage is to determine how the user can perform this task, and how it fits into the overall user experience. Please read the full breakdown of my process development in the prototype process page, but the simplified idea is that I wanted to implement a form of tapping mechanic that user can use to tap onto an RFID/NFC reader, which would then trigger the printer to print. And I set about purchasing both the NFC chips and also the reader.

One of the tricky parts is that the reader I bought, though worked quite well as a card reader, for my purposes, it presented an issue where when I tap the NFC chips, the reader immediate read and 'Enter' the input into whatever is open on the computer. There's no buffer in between the input and the enter.

So to combat this, I created a web portal that would be on when the reader enters the input, and it would take this input and trigger the printer.
